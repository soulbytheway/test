## 3. Розробка діаграм варіантів використання

Нижче представлено UML-діаграму варіантів використання, побудовану на основі узагальнених акторів і варіантів використання, наведених у таблиці 2.

```mermaid
usecaseDiagram
    actor "Користувач системи" as User
    actor "Рецензент" as Reviewer
    actor "Зовнішні джерела інформації" as External
    actor "Система" as System

    rectangle "Система Zotero" {
        (Додавання джерела)
        (Імпорт джерела)
        (Створення тематичних колекцій)
        (Анотування джерел)
        (Форматування бібліографії)
        (Створення цитат)
        (Перевірка бібліографії)
        (Експертиза джерел)
        (Надання метаданих)
        (Автоматичне оновлення метаданих)
        (Синхронізація бібліотеки)
        (Резервне копіювання)
    }

    %% Зв'язки користувача
    User --> (Додавання джерела)
    User --> (Імпорт джерела)
    User --> (Створення тематичних колекцій)
    User --> (Анотування джерел)
    User --> (Форматування бібліографії)
    User --> (Створення цитат)

    %% Зв'язки рецензента
    Reviewer --> (Перевірка бібліографії)
    Reviewer --> (Експертиза джерел)

    %% Зв'язки зовнішніх джерел
    External --> (Надання метаданих)
    External --> (Імпорт джерела)

    %% Зв'язки системи
    System --> (Автоматичне оновлення метаданих)
    System --> (Синхронізація бібліотеки)
    System --> (Резервне копіювання)

    %% Внутрішні залежності між прецедентами
    (Імпорт джерела) --> (Автоматичне оновлення метаданих)
    (Додавання джерела) --> (Синхронізація бібліотеки)
    (Створення тематичних колекцій) --> (Резервне копіювання)
